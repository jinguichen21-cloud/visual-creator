# AI视觉工坊

参考图搜索 → 文生图 / 图生图 / 海报生成 → 迭代优化 → 交付成品。

## MCP 服务配置

本技能依赖以下 MCP 服务，运行时**必须**先读取 `mcp-config.json` 获取服务 URL：

| 环境变量名 | MCP 服务 | mcpId | 用途 |
|------------|----------|-------|------|
| `$MCP_T2I_URL` | 通义万相-文生图 | 1064 | 文字描述生成图片（wan2.2，异步，写实/摄影风格） |
| `$MCP_T2I_V25_URL` | 通义万相-文生图wan2.5 | 2429 | 文字描述生成图片（wan2.5，新版模型） |
| `$MCP_I2I_URL` | 通义万相-图生图 | 2416 | 基于参考图编辑/多图融合（wan2.5，最多3张） |
| `$MCP_POSTER_URL` | 创意海报生成 | 9436 | 自然语言生成高质量海报（qwen-image-max） |
| `$MCP_IMG_SEARCH_URL` | 小宿-图片搜索 | 9666 | 搜索参考图片素材 |

### 配置读取方式

**约束：调用方 agent 必须在执行任何 MCP 调用前，先读取 mcp-config.json 文件**

```bash
python3 scripts/call_mcp.py list "$MCP_T2I_URL"
```

## 创作模式

### 模式一：文生图

用户用文字描述想要的图片，AI 生成。

**适用场景**：公众号配图、社交媒体素材、概念设计、产品展示、写实摄影效果

**两个模型选择策略**：
| 模型 | 环境变量 | 适用场景 | 特点 |
|------|----------|----------|------|
| wan2.2-t2i-flash | `$MCP_T2I_URL` | 写实场景、摄影风格 | 异步任务，需轮询结果 |
| wan2.5-t2i-preview | `$MCP_T2I_V25_URL` | 通用场景、更高质量 | 新版模型 |

- 用户要求写实/摄影风格 → 优先 wan2.2
- 用户要求插画/创意/通用 → 优先 wan2.5
- 用户未指定 → 默认 wan2.5

#### wan2.2 文生图（异步）

```bash
# 1. 提交生图任务
python3 scripts/call_mcp.py call "$MCP_T2I_URL" start_image_generation \
  --params '{"prompt": "<优化后的英文/中文提示词>", "width": 1024, "height": 1024}'

# 2. 等待 10 秒后查询结果（必须等待！）
python3 scripts/call_mcp.py call "$MCP_T2I_URL" get_image_generation_result \
  --params '{"taskId": "<返回的taskId>"}'
```

如果返回状态为运行中，继续轮询直到成功或失败。

#### wan2.5 文生图

```bash
python3 scripts/call_mcp.py call "$MCP_T2I_V25_URL" gen_image_by_text \
  --params '{"prompt": "<优化后的提示词>"}'
```

### 模式二：图生图

用户提供参考图 + 文字描述，AI 基于参考图编辑或融合。

**适用场景**：风格迁移、图片编辑、多图合成、基于参考图的再创作

```bash
# 1. 提交图生图任务（支持 1-3 张参考图）
python3 scripts/call_mcp.py call "$MCP_I2I_URL" start_image_generation \
  --params '{
    "prompt": "<编辑描述>",
    "imageUrls": ["<图片URL1>", "<图片URL2>"]
  }'

# 2. 等待后查询结果
python3 scripts/call_mcp.py call "$MCP_I2I_URL" get_image_generation_result \
  --params '{"taskId": "<返回的taskId>"}'
```

---

### 模式三：创意海报

用户用自然语言描述海报需求，AI 生成包含文字排版的完整海报。

**适用场景**：活动海报、促销海报、节日海报、品牌宣传、社交媒体封面

```bash
python3 scripts/call_mcp.py call "$MCP_POSTER_URL" generation_poster_by_text \
  --params '{"prompt": "<海报描述，包含主题、文案、风格、品牌色等>"}'
```

**提示词建议**：描述中应包含以下要素：
- 海报主题/用途
- 需要展示的文案内容
- 视觉风格（简约/复古/科技/国潮等）
- 品牌色/主色调（如有）
- 尺寸偏好（横版/竖版/方形）

---

### 模式四：参考图搜索

用户描述想要的风格，先搜索参考图，再决定用哪种模式创作。

```bash
python3 scripts/call_mcp.py call "$MCP_IMG_SEARCH_URL" search_images \
  --params '{"query": "<风格/场景描述>"}'
```

搜索到的图片可以直接用作图生图的参考输入。

## 工作流程

### 第一步：理解创作需求

收到用户请求后，判断：

1. **创作类型**：文生图 / 图生图 / 海报 / 不确定
2. **风格要求**：写实、插画、扁平、3D、国潮、极简等
3. **用途场景**：社交媒体、公众号、PPT、印刷品、电商等
4. **尺寸需求**：横版(16:9)、竖版(9:16)、方形(1:1)、自定义

如果用户需求模糊，先搜索参考图帮助确认方向。

### 第二步：优化提示词

用户的描述往往不够详细。AI 需要将用户意图转化为高质量提示词：

**提示词优化原则**：
- 补充画面细节：光线、构图、色调、材质
- 指定风格关键词：photorealistic / flat design / watercolor / cyberpunk
- 添加质量修饰：high quality, detailed, 8k, professional
- 对于海报：明确文案内容、排版方向、品牌色

**示例**：
- 用户说："帮我画个猫" → 优化为："A fluffy orange tabby cat sitting on a windowsill, warm afternoon sunlight, bokeh background, photorealistic, 8k, Canon EOS R5 style"
- 用户说："做个双11海报" → 优化为："双十一大促海报，主标题'全场5折起'，副标题'11.11狂欢盛典'，红色金色为主色调，电商风格，竖版，底部留白放商品图"

### 第三步：执行生图

根据第一步判断的类型，调用对应 MCP 服务。

**路由规则**：

| 用户需求 | 路由到 |
|----------|--------|
| "画一张/生成图片" + 纯文字描述 | 文生图（wan2.5 优先） |
| "写实/摄影风格" + 纯文字描述 | 文生图（wan2.2 优先） |
| 提供了参考图 + "改一下/换风格" | 图生图 |
| 提供了多张图 + "融合/合成" | 图生图（多图融合） |
| "做海报/banner/封面" + 含文案需求 | 创意海报 |
| "找参考/有没有类似的" | 先搜图，再决定 |

### 第四步：展示结果与迭代

生成结果后：
1. 展示生成的图片 URL
2. 询问是否满意
3. 如需调整：修改提示词重新生成，或用图生图基于当前结果微调

```
- "基于这张改一下" → 切换到图生图模式微调

```

## 场景示例

### 场景一：公众号配图
用户："帮我画一张关于远程办公的配图"
→ wan2.5 文生图 → 生成 → 展示 → 迭代

### 场景二：基于参考图再创作
用户："参考这张图，帮我改成科技风格" + 附图
→ 图生图 → 生成 → 展示

### 场景三：活动海报
用户："做一张年终总结大会的海报，主题'2025再出发'"
→ 创意海报 → 优化提示词（补充风格、色调、排版） → 生成

### 场景四：不确定要什么
用户："帮我做个好看的封面"
→ 先搜参考图 → 用户选方向 → 文生图或海报

---

## 注意事项

- wan2.2 文生图是异步任务，提交后**必须等待至少10秒**再查询结果
- 图生图最多支持 3 张参考图
- 创意海报的提示词中应尽量包含完整文案内容，避免后续修改
- 输出分辨率：文生图宽高均在 [512, 1440] 像素之间
- 如某个 MCP 调用失败，尝试用其他模式替代（如海报失败可用文生图）
- 提示词优化是关键步骤，不要直接把用户原话当提示词使用
